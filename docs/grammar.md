# 🍁枫叶指令宏

 🚀 **枫叶指令宏——让自动化变得简单！无需编程基础，通过简单指令即可实现自动化操作。**

---

### 1. 📝 核心语法规范

---

#### 1.1 输入法规则
- 所有指令、参数必须使用小写英文字母输入，禁止中文全角字符。

---
#### 1.2 书写规则
- 指令与指令之间不能有空格、退格符，所有内容必须紧贴书写。

---
#### 1.3 左侧对齐规则
- 每一行代码都必须从行首开始，禁止任何形式的缩进（空格、Tab开头）。

- 任何有缩进的行都会被判定为语法错误，无法执行。

---
#### 1.4 注释规则
- 以英文分号;开头的行均为注释，执行时完全忽略。

- 注释必须独占一行，不可写在指令同行末尾。


```
 ; 这是一条注释.运行时不会运行这一条
```
---
#### 1.5 分隔符规则
- 指令分隔符：指令与参数之间用反斜杠\分隔。即使没有参数，指令末尾也必须保留\。

```
按键\a\
鼠标\左键单击\
延迟\500,1000\
```

- 参数分隔符：多个参数之间用英文逗号,分隔。

```
鼠标移动\777,574\
```

#### 1.6 变量调用规则
- 使用@变量名的形式引用已定义的变量值。

```
弹窗提示\@超时\
```

---

#### 1.7 指令基本规则

---

- **基本指令**：形式为 指令\参数\ 或 指令\参数,参数\，可能带多个参数（独立可执行的基础指令）。

- **判断指令**：以如果\ 开头，后跟一个或多个条件指令，条件指令之间可通过条件连接指令 (与\ 或\) 衔接，最终通过分支指令（成功\、失败\）处理判断结果。

- **条件指令**：比较\、坐标找\、范围找色\、识图\（判断指令的核心条件单元）。

- **条件连接指令**：与\、或\（仅用于连接判断指令内的多个同类型条件指令）。

- **分支指令**：成功\、失败\（仅依附判断指令存在），其后可接多个基本指令，禁止接判断指令或结构体；循环特殊指令可接在分支指令的指令序列末尾。

- **结构体**：循环结构体、模块结构体（包含多组指令的复合逻辑单元）。

- **循环控制指令**：（仅在循环结构内使用）：返回循环\、终止循环\，需独立成行，可作为分支指令末尾的收尾指令。

---
### 2. ⏱️ 延迟指令 (基本指令)
- 格式：延迟\最小时长,最大时长\

- 功能：在最小和最大时长之间随机等待（毫秒）。如果两个值相同，则为固定等待。

- 参数：可传变量。

- 示例：

```
延迟\100,300\       ; 随机等待100~300毫秒
延迟\500,500\       ; 固定等待500毫秒
```

### 5. ⚖️ 条件判断指令 (判断指令)

---

- 结构：
```
如果\条件指令\参数...\
成功\基本指令1\基本指令2\...   ; 条件成立时执行
失败\基本指令a\基本指令b\...   ; 条件不成立时执行
```
- 条件指令可以组合使用与\、或\连接多个条件。
- 成功\和失败\块后可接多个基本指令，但不能嵌套判断指令或结构体。

---

####  **- 支持的条件指令 -**

---

#### 5.1 比较 (条件指令) 

---

- **格式**：比较\值1,比较符,值2,变量名\

- 比较符：等于、不等于、大于、小于、大于等于、小于等于、字符包含

- 比较结果存入变量（1=真，0=假）

- 示例 (可复制测试)：

```
变量\血量,20\
如果\比较\@血量,小于,30,低血量\
成功\弹窗提示\血量过低！\
失败\弹窗提示\血量安全\
```

---

#### 5.2 坐标找色 (条件指令) 

---

- **格式**：坐标找色\X坐标,Y坐标,颜色值,变量名\

- 获取指定像素点的颜色值，存入变量（十六进制字符串）。

- 示例 (可复制测试)：

```
如果\坐标找色\455,580,0xAE7222,色值结果\
成功\弹窗提示\找到颜色@色值结果\
失败\弹窗提示\颜色不匹配\
```
---

#### 5.3 范围找色 (条件指令) 

---

- **格式**：范围找色\X1,Y1,X2,Y2,颜色值,渐变值,结果变量,坐标X,坐标Y\

- 在矩形区域内查找颜色，支持渐变容差（0~255）。

- 结果变量：1=找到，0=没找到；坐标X、Y变量存储第一个匹配点的坐标。

- 示例 (可复制测试)：

```
如果\范围找色\100,100,200,200,0xFFFFFF,0,找色结果,找色X,找色Y\
成功\鼠标移动\@找色X,@找色Y\
失败\弹窗提示\未找到白色\
```

---

#### 5.4 识图 (条件指令) 

---
- 格式：识图\X1,Y1,X2,Y2,容错,<识图标记>,标记结果,识图数量,识图X,识图Y,模式,时间\

- X1,Y1,X2,Y2：搜索区域坐标。

- 容错：0.0~1.0，0.1表示10%容差。

- <识图标记>：抓图工具保存的标签名。

- 标记结果：保存找到的标签（通常与标记相同）。

- 识图数量：保存找到的图像个数。

- 识图X,识图Y：保存第一个找到的图像中心坐标。此外系统会自动生成识图X1,识图Y1...变量存储多个坐标。

- 模式：1（识别一次，默认）、2（等待出现）、3（等待消失）。

- 时间：等待模式下的超时秒数（0表示无限等待）。

---
- 示例（识别一次模式）：

```
如果\识图\100,200,300,400,0.1,<开始游戏>,标记结果,识图数量,识图X,识图Y\
成功\识图移动\0,0\鼠标\左键单击\
失败\弹窗提示\识别失败\
```

---
- 示例（等待出现模式）：

```
如果\识图\100,200,300,400,0.1,<开始游戏>,标记结果,识图数量,识图X,识图Y,等待出现,3\
成功\识图移动\0,0\鼠标\左键单击\
失败\弹窗提示\识别失败\
```
---
- 示例（等待消失模式）：

```
如果\识图\100,200,300,400,0.1,<开始游戏>,标记结果,识图数量,识图X,识图Y,等待消失,3\
成功\识图移动\0,0\鼠标\左键单击\
失败\弹窗提示\识别失败\
```

---

#### 5.5 与/或 (条件连接指令)

---

- 使用与\和或\连接多个条件指令，实现复合判断。

- 示例（与）(可复制测试)：

如果\比较\@a,等于,1,结果1\或\比较\@b,等于,2,结果2\

#### 6.1 循环结构体
---
**循环结构体用于重复执行一段指令序列。**

**重要**：循环结构体**不可嵌套**。若需实现循环嵌套，请通过模块调用来完成（详见 6.2 模块结构体）。**

---
- 结构：

```
循环\次数\          ; 次数为 0 表示无限循环
    ...             ; 循环体内的指令（可包含基本指令、判断指令等）
结束循环\           ; 循环结束标记
```

- 作用：反复执行循环体内的所有指令，直到循环次数用尽或被循环控制指令提前终止。

- 简单示例(可复制测试)：

```
; 左键连点10下
; 
循环\10\
鼠标\左键单击\
结束循环\
```


- **循环控制指令**

	在循环体内，可以使用以下两条特殊指令来控制循环流程：

  - **返回循环**：立即跳转到循环开始，开始下一次循环（后续指令不再执行）。

  - **终止循环**：立即退出当前循环，继续执行 结束循环\ 之后的指令。
  
**使用规则**：

  - 循环控制指令**必须独立成行**，即该行只能包含这一条指令（允许带有分支指令前缀，如 成功\终止循环\）。
  - 在分支指令（成功\ 或 失败\）的指令序列中，循环控制指令必须作为该分支的最后一行，且该行不能与其他指令（如基本指令）混合书写。
  - 循环控制指令必须独占一行。
  
  - 简单示例(可复制测试)：
  
```
; 一个简单计数器  (屏幕左上角查看悬浮提示)
; 
变量\计数,0\
; 
循环\0\
悬浮提示\@计数,0,0,1\
如果\比较\@计数,大于等于,10,结果\
成功\终止循环\
失败\运算\@计数,加,1,计数\
延迟\1000,1000\
结束循环\
```

```
; 一个简单倒计时  (屏幕左上角查看悬浮提示)
; 
变量\倒计时,5\                ; 设置初始值
; 
循环\5\                       ; 循环5次
悬浮提示\剩余 @倒计时 秒,0,0,1\
延迟\1000,1000\           ; 等待1秒
运算\@倒计时,减,1,倒计时\ ; 倒计时减1
结束循环\
; 
弹窗提示\时间到！\
```

- 正确示例：

```
循环\10\
如果\识图\0,0,0,0,0.1,<数据名>,标记结果,识图数量,识图X,识图Y\
成功\识图移动\0,0\延迟\500,1000\鼠标\左键单击\
成功\结束循环\		 ; 成功找到后退出循环（独立成行）
失败\弹窗提示\失败!\
失败\返回循环\		 ; 失败后开始下一次循环（独立成行）
结束循环\
```

- 错误示例：

```
循环\10\
如果\识图\0,0,0,0,0.1,<数据名>,标记结果,识图数量,识图X,识图Y\
成功\识图移动\0,0\延迟\500,1000\鼠标\左键单击\结束循环\		 ;  这里不操作直接结束循环
失败\弹窗提示\失败!\返回循环\		                             ;  这里不弹窗直接返回循环
结束循环\
```

**注意**：在需要执行的分支中，每一行必须以 成功\ 或 失败\ 开头。循环控制指令所在行只能有该指令（带分支前缀），不能与其他指令同行。错误示例中将控制指令与基本指令写在同一行（如 成功\识图移动\0,0\延迟\...\结束循环\）是不允许的。

#### 6.2 模块结构体
---
**模块结构体用于封装一段可复用的指令序列，实现代码的模块化。**

---

- 结构：

```
模块定义\模块名\
    ...                 ; 模块内的指令（可包含任意指令，但不能嵌套模块定义）
结束模块定义\
```




- **特性**

  - 模块内可以调用其他模块，但需注意避免递归调用导致死循环。

  - 模块内可包含循环、判断等任何结构体，但不能再定义新模块（即模块定义不可嵌套）。

  - 模块调用会同步执行，即调用后等待模块执行完毕才继续后续指令。
- 调用格式：

```
模块\模块名\            ; 基本指令，用于执行已定义的模块
```
- 示例：

```
模块定义\登录游戏\
鼠标移动\500,300\
鼠标\左键单击\
延迟\1000,1000\
按键\Enter\
结束模块定义\
; 
模块\登录游戏\       ; 调用模块

```

- “如果-否则如果-否则”结构示例：

```
模块定义\否则如果\
如果\比较\@a,等于,2,比较结果2\
成功\弹窗提示\a=2\
失败\弹窗提示\a既不是1也不是2\
结束模块定义\
; 
变量\a,0\
如果\比较\@a,等于,1,比较结果1\
成功\弹窗提示\a=1\
失败\模块\否则如果\
```


- 模块嵌套调用示例 (可复制测试)：：

```
模块定义\a\
循环\2\					   ; 循环2次
按键\a\                    ; 每次按一下ａ
结束循环\
结束模块定义\
; 
循环\3\
模块\a\                ;  调用3次模块
结束循环\
```


- 模块嵌套调用示例 (可复制测试)：：

```
; 定义模块 B
模块定义\模块B\
弹窗提示\模块B 被调用\
结束模块定义\
; 
; 定义模块 A
模块定义\模块A\
弹窗提示\模块A 开始执行\
模块\模块B\               ; 调用模块 B
弹窗提示\模块A 执行结束\
结束模块定义\
; 
; 主程序：调用模块 A
模块\模块A\
```


---
- 定时调用格式：

```
模块定时\模块名,毫秒,模式\      ; 基本指令，用于定时执行已定义的模块
```
---
- 格式：模块定时\模块名,毫秒,模式\

- 模式：重复（周期调用）、一次（单次调用）、关闭（取消定时）。

- 功能：在指定延迟后异步调用模块，不阻塞主线程。

- 示例 (可复制测试)：

```
模块定义\提醒\
弹窗提示\时间到！\
结束模块定义\
; 
模块定时\提醒,5000,一次\   ; 5秒后弹窗
延迟\10000,10000\          ; 主线程继续执行
```
