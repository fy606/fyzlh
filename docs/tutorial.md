#### 1.1 第一个脚本：自动打开记事本并输入文字

---

- 目标：自动启动 Windows 记事本，等待窗口打开，然后输入一段文字 “Hello World”。

- 核心指令：

  - 运行程序\：启动记事本程序。

  - 延迟\：等待窗口加载。

  - 激活窗口\：将记事本窗口置前。

  - 按键\：模拟键盘输入。
  
  - 文本输入\：模拟文本输入。

  - 完整脚本：

```
; 第一个脚本：自动打开记事本并输入 Hello World
运行程序\notepad.exe\
延迟\1000,1500\          ; 等待记事本启动
激活窗口\记事本\          ; 激活记事本窗口（窗口标题包含“记事本”即可）
延迟\500,500\            ; 确保窗口完全激活
按键\H\                   ; 逐个输入字母
按键\e\
按键\l\
按键\l\
按键\o\
按键\Space\               ; 输入空格
按键\W\
按键\o\
按键\r\
按键\l\
按键\d\
; 输入完成
```

```
; 第一个脚本：自动打开记事本并输入 Hello World
运行程序\notepad.exe\
延迟\1000,1500\          ; 等待记事本启动
激活窗口\记事本\          ; 激活记事本窗口（窗口标题包含“记事本”即可）
延迟\500,500\            ; 确保窗口完全激活
文本输入\Hello World\
; 输入完成
```

- 执行说明：

- 如果记事本启动较慢，可以适当增加延迟范围。

- 激活窗口时，如果系统中有多个包含“记事本”的窗口，可能会激活错误，可通过更精确的窗口标题（如“无标题 - 记事本”）避免。

#### 1.2 第二个脚本：定时检测窗口并弹窗提醒
---

- 目标：每 5 秒检测一次“微信”窗口是否存在，如果存在则弹出提示并停止检测。

- 实现思路：使用模块定时\指令创建一个周期任务，在模块内判断窗口是否存在，存在时弹窗并关闭定时器。

- 完整脚本：

```
; 定义检测模块
模块定义\窗口检测\
判断窗口\微信,存在,窗口存在结果\   ; 判断微信窗口是否存在
如果\比较\@窗口存在结果,等于,1,比较结果\
成功\弹窗提示\微信已运行！\模块定时\窗口检测,0,关闭\   ; 存在则弹窗并关闭定时
失败\悬浮提示\未检测到微信,100,100,1\                 ; 不存在时显示悬浮提示
结束模块定义\
; 
; 启动定时检测，每5秒执行一次
模块定时\窗口检测,5000,重复\
; 
; 主线程保持运行（例如持续10秒，实际脚本可能无限循环）
延迟\10000,10000\
; 10秒后主线程结束，定时模块自动清除
```
执行说明：

- 判断窗口\的第二个参数是“存在”，检测窗口是否存在；也可以改为“激活”检测窗口是否处于激活状态。

- 定时模块一旦启动，会独立于主线程运行，主线程结束时所有定时任务自动终止。

- 弹窗后关闭定时器，避免重复弹窗。

#### 1.3 第三个脚本：图像识别点击广告关闭按钮
---

- 目标：在屏幕指定区域查找“关闭”按钮图像（例如广告弹窗的 X 按钮），找到后点击它。

- 前提：已使用抓图工具截取关闭按钮图像并命名为<X按钮>。

- 实现思路：使用识图指令的“等待出现”模式，循环等待按钮出现，出现后移动鼠标点击并退出循环。

- 完整脚本：

```
; 定义搜索区域（假设广告弹窗通常出现在屏幕中央）
变量\区域左,600\
变量\区域上,400\
变量\区域右,800\
变量\区域下,600\
; 
; 循环等待关闭按钮出现
循环\0\
如果\识图\@区域左,@区域上,@区域右,@区域下,0.1,<X按钮>,标记,数量,X,Y,等待出现,10\
成功\识图移动\0,0\        ; 移动到按钮中心（识图移动会自动使用上一次识别到的坐标）
成功\鼠标\左键单击\
成功\延迟\500,500\
成功\终止循环\            ; 点击后退出循环
失败\延迟\1000,1000\      ; 没找到则等待1秒后继续查找
结束循环\
; 
; 点击完成
悬浮提示\广告已关闭,0,0,1\
弹窗提示\广告已关闭\
```

#### 1.4 第四个脚本：鼠标移动至两张图片中心点
---

- 目标：在屏幕上同时找到图像 a 和图像 b，计算两者在 X 轴方向的中间位置，并将鼠标移动到该中间点（Y 轴坐标沿用图像 a 的 Y 坐标）。

- 前提：已使用抓图工具截取需要识别的两个目标图像，并分别命名为 a 和 b。

- 实现思路：

	- 同时识别屏幕上的图像 a 和 b，获取 a 的 X 坐标 X1、Y 坐标 识图Y，以及 b 的 X 坐标 X2。
	
	- 计算 X1 与 X2 的距离，取距离的一半得到 “半距”。
	
	- 用 X1 加上 “半距”，得到两个图像在 X 轴的中间坐标。
	
	- 将鼠标移动到（中间 X 坐标，识图Y）的位置。

- 完整脚本：

```
;  移动鼠标到 a b 的图像 中间
如果\识图\0,0,0,0,0.1,<a>,标记结果,识图数量,X1,识图Y\与\识图\0,0,0,0,0.1,<b>,标记结果,识图数量,X2,识图Y\
成功\运算\@X2,减,@X1,距离\
成功\运算\@距离,除,2,半\
成功\运算\@X1,加,@半,中间\
成功\鼠标移动\@中间,@识图Y\
```

#### 2.1 脚本调试技巧

---

- 利用弹窗/悬浮提示输出变量

  - 弹窗提示\会阻塞脚本，适合查看关键节点的变量值。

  - 悬浮提示\不阻塞，适合持续监控变量变化（可指定不同编号同时显示多个值）。
  
- 示例：输出循环计数

```
变量\计数,0\
循环\0\
运算\@计数,加,1,计数\
悬浮提示\当前计数:@计数,100,100,1\
如果\比较\@计数,大于,5,结果\
成功\终止循环\
失败\延迟\500,500\
结束循环\
```

- **断点思路**

  - 在疑似问题位置前插入弹窗提示\暂停\，脚本执行到此会暂停，点击确定后继续。相当于设置断点。

  - 结合获取鼠标\、获取时间\等指令输出当前环境信息。


- **日志记录**

  - 可以查看日志检测
  
  - 查看错误日志定位问题行。

#### 2.2 错误处理与容错设计

---

- 利用失败\分支实现重试机制

- 在可能失败的操作（如识图、找色）后，通过循环和计数实现有限次重试。

- 示例：识图点击重试最多3次

```
变量\重试次数,0\
循环\0\
如果\比较\@重试次数,大于等于,3,结果\
成功\弹窗提示\重试超时，退出\
成功\终止循环\
失败\识图\0,0,1000,800,0.1,<button>,标记,数量,x,y,1,0\
成功\识图移动\0,0\鼠标\左键单击\
成功\终止循环\
失败\运算\@重试次数,加,1,重试次数\延迟\1000,1000\
结束循环\
```

- 超时退出设计

- 在循环开始前记录起始时间（使用获取时间\转换为秒数计算），循环内判断是否超时。

```
获取时间\年,月,日,时,分,秒\
变量\开始秒,0\
运算\@时,乘,3600,开始秒\
运算\@开始秒,加,@分乘60,开始秒\
运算\@开始秒,加,@秒,开始秒\   ; 将当前时间转换为总秒数
; 
循环\0\
获取时间\年,月,日,时,分,秒\
变量\当前秒,0\
运算\@时,乘,3600,当前秒\
运算\@当前秒,加,@分乘60,当前秒\
运算\@当前秒,加,@秒,当前秒\
运算\@当前秒,减,@开始秒,耗时\
如果\比较\@耗时,大于,30,结果\   ; 超时30秒
成功\弹窗提示\操作超时\
成功\终止循环\
失败\...正常操作...
结束循环\

```
